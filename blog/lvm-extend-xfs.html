LVM Extend Sonrası XFS Büyümedi: Çözüm Adımları

Production VMware ortamında LVM genişlettikten sonra XFS dosya sistemi neden büyümedi ve bunu canlı sistem durmadan nasıl çözdüm.

Problem Senaryosu

Production ortamında çalışan bir VMware sanal makinesinde disk alanı dolmak üzereydi. VMware tarafında diski genişlettim ve LVM logical volume'ü büyüttüm. Ancak df -h komutu hala eski boyutu gösteriyordu.

İşte karşılaştığım durum:

# Disk genişletme sonrası
lsblk
NAME                MAJ:MIN RM  SIZE RO TYPE MOUNTPOINT
sda                   8:0    0  200G  0 disk 
├─sda1                8:1    0    1G  0 part /boot
└─sda2                8:2    0  199G  0 part 
  └─vgdata-lvdata   253:0    0  199G  0 lvm  /data

# Ancak dosya sistemi hala eski boyutta
df -h /data
Filesystem                Size  Used Avail Use% Mounted on
/dev/mapper/vgdata-lvdata  99G   95G  4.0G  96% /data

Yanlış Denenen Çözümler

İlk olarak klasik resize2fs komutunu denedim, ancak XFS dosya sistemi için bu komut çalışmıyor. XFS için doğru komut xfs_growfs. 

# Bu hata verdi
resize2fs /dev/mapper/vgdata-lvdata
resize2fs: Bad magic number in super-block while trying to open /dev/mapper/vgdata-lvdata

Sonra xfs_growfs komutunu denedim ama yine hata aldım: 

# Bu da çalışmadı
xfs_growfs /dev/mapper/vgdata-lvdata
xfs_growfs: /dev/mapper/vgdata-lvdata is not a mounted XFS filesystem

Kesin Çözüm Adımları

Sorun şuydu: xfs_growfs komutu cihaz adı değil, mount point bekliyor. İşte doğru adımlar:

1. Mevcut Durumu Kontrol Et

# LVM boyutunu kontrol et
lvdisplay /dev/vgdata/lvdata
  LV Size                199.00 GiB

# Dosya sistemi boyutunu kontrol et
df -h /data
Filesystem                Size  Used Avail Use% Mounted on
/dev/mapper/vgdata-lvdata  99G   95G  4.0G  96% /data

# Mount point'i kontrol et
mount | grep /data
/dev/mapper/vgdata-lvdata on /data type xfs (rw,relatime)

2. XFS Dosya Sistemini Büyüt

# Doğru komut: mount point kullan, cihaz adı değil
xfs_growfs /data

meta-data=/dev/mapper/vgdata-lvdata isize=512    agcount=4, agsize=6553600 blks
         =                       sectsz=512   attr=2, projid32bit=1
data     =                       bsize=4096   blocks=26214400, imaxpct=25
         =                       sunit=0      swidth=0 blks
naming   =version 2              bsize=4096   ascii-ci=0 ftype=1
log      =internal               bsize=4096   blocks=12800, version=2
realtime =none                   extsz=4096   blocks=0, rtextents=0
data blocks changed from 26214400 to 52428800

3. Sonucu Doğrula

# Dosya sistemi boyutunu tekrar kontrol et
df -h /data
Filesystem                Size  Used Avail Use% Mounted on
/dev/mapper/vgdata-lvdata 199G   95G  104G  48% /data

# Başarılı! Dosya sistemi büyüdü

Dikkat Edilmesi Gereken Noktalar

  • XFS için mount point kullan: xfs_growfs komutu cihaz adı değil, mount point bekler.
  • Sistem çalışırken yapılabilir: XFS dosya sistemini unmount etmeden büyütebilirsiniz.
  • Küçültme yapılamaz: XFS dosya sistemi sadece büyütülebilir, küçültülemez.
  • Yedek alın: Production ortamında her zaman önce yedek alın.
  • LVM snapshot: Kritik sistemlerde işlem öncesi LVM snapshot oluşturun.

Alternatif: LVM Extend ile Birlikte

Eğer LVM extend işlemini henüz yapmadıysanız, -r parametresi ile dosya sistemini otomatik büyütebilirsiniz: 

# LVM extend ve dosya sistemi büyütme tek komutta
lvextend -r -L +100G /dev/vgdata/lvdata

# veya tüm boş alanı kullan
lvextend -r -l +100%FREE /dev/vgdata/lvdata

Sonuç

XFS dosya sistemi ile çalışırken resize2fs yerine xfs_growfs kullanmak ve mount point belirtmek kritik. Bu basit detay, production ortamında saatler sürebilecek bir sorunu dakikalar içinde çözmenizi sağlar.

Benzer bir durumla karşılaşırsanız, önce dosya sistemi tipini kontrol edin (mount | grep /data) ve doğru aracı kullanın.
